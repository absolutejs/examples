OAuth2 / OIDC Provider

This example app issues OAuth2/OIDC tokens to client apps (RPs) configured via Dynamic Client Registration (RFC 7591). The two buttons below fetch the public discovery document + JWKS — the standard "anyone can read this" endpoints any RP would call to bootstrap.

Discovery

GET /.well-known/openid-configuration


        

        

          
JWKS

          
GET /oauth2/jwks

          
          

        

        

          
Register a client (DCR)

          

            RFC 7591 — POST your client metadata to
            /oauth2/register:
          

          
curl -X POST /oauth2/register \
  -H 'content-type: application/json' \
  -d '{
    "client_name": "Acme RP",
    "redirect_uris": ["https://acme.example/cb"],
    "grant_types": ["authorization_code", "refresh_token"]
  }'
