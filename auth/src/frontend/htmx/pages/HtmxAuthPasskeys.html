Passkeys

WebAuthn passkeys require a full browser ceremony (navigator.credentials.{get, create}), so they can't be driven by pure server-rendered HTML. Visit the React, Vue, Svelte, Angular, or HTML tabs to see the conditional-UI autofill + upgrade-prompt flows working end-to-end (they all use runConditionalAuthentication + runPasskeyRegistration from @absolutejs/auth/client).

A future package PR could add HTMX wrappers around the WebAuthn handshake (returning HTML fragments for each step), but until then this page links out to the framework versions.
